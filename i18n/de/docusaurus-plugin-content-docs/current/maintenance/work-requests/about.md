# Arbeitsanforderungen

Eine **Arbeitsanforderung** ist eine formale Einreichung zur Erstellung eines Arbeitsauftrags für Wartungsarbeiten, Reparaturen oder Anlagenprobleme.

| Benutzertyp | Wie sie einreichen |
|---|---|
| **Interne Benutzer** | Personal mit eingeschränktem Zugriff (Anforderer) erstellt Arbeitsanforderungen direkt in der Anwendung. |
| **Externe Benutzer** | Administratoren können webbasierte **Gastportale** konfigurieren, über die Personen ohne Konto Anforderungen einreichen können. |

## Der Genehmigungsprozess

Wartungsverantwortliche oder Administratoren müssen alle eingehenden Anforderungen überprüfen, um sie zu **genehmigen** oder **abzulehnen**.

- **Konvertierung**: Nach der Genehmigung wird die Arbeitsanforderung sofort in einen aktiven Arbeitsauftrag umgewandelt.

:::tip Wann Sie Arbeitsanforderungen überspringen sollten
Nur begrenzte „Anforderer"-Benutzer sind verpflichtet, diesen Ablauf zu verwenden. **Administratoren und Benutzer mit vollem Zugriff** können Arbeitsaufträge direkt erstellen.
:::

## Gastportale

Ein **Gastportal** ist ein eigenständiges Webformular, das es jedem ermöglicht, ein Problem zu melden, **ohne ein Konto zu benötigen**. Jede Person mit der dedizierten URL oder dem QR-Code des Portals kann sofort eine Arbeitsanforderung erstellen.
