# طلبات العمل

**طلب العمل** هو تقديم رسمي لإنشاء أمر عمل للصيانة أو الإصلاحات أو مشاكل المرافق.

| نوع المستخدم | كيفية التقديم |
|---|---|
| **المستخدمون الداخليون** | يُنشئ الموظفون ذوو الصلاحيات المحدودة (الطالبون) طلبات العمل مباشرة من التطبيق الرئيسي. |
| **المستخدمون الخارجيون** | يمكن للمسؤولين إعداد **بوابات ضيف** عبر الويب حيث يستطيع الأفراد بدون حساب تقديم طلبات. |

## عملية الموافقة

يجب على مديري الصيانة أو المسؤولين مراجعة جميع الطلبات الواردة إما **لقبولها** أو **رفضها**.

- **التحويل**: عند القبول، يُحوَّل طلب العمل فوراً إلى أمر عمل نشط.

:::tip متى تتجاوز طلب العمل
فقط مستخدمو «الطالب» المحدودون مطالبون باستخدام هذا سير العمل. **مسؤولو النظام والمستخدمون ذوو الوصول الكامل** يمكنهم تجاوز هذا الصندوق وإنشاء أوامر عمل مباشرة.
:::

## وحدة طلبات العمل

تُعدّ لوحة طلبات العمل صندوق البريد المركزي لإدارة التذاكر:

- **للطالبين**: هنا يُقدّمون تذاكر جديدة ويتتبّعون حالة مشاكلهم الموردة.
- **للمسؤولين**: هذا هو مركز الفرز حيث يعرضون ويُصفّون ويقبلون أو يرفضون جميع التقديمات المعلّقة.

## بوابات الضيف

**بوابة الضيف** هي نموذج ويب مستقل يسمح لأي شخص بالإبلاغ عن مشكلة **دون الحاجة إلى حساب أو بيانات دخول**. يمكن لأي شخص لديه الرابط أو يمسح رمز QR الخاص بالبوابة إنشاء طلب عمل فوراً.

يستطيع المسؤولون:
- تهيئة بوابات ضيف متعددة
- استهدافها لآلات أو مناطق مرافق محددة
- وضع رموز QR مباشرة على المعدات
