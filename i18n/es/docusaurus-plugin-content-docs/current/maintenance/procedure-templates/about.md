# Plantillas de Procedimientos

Las Plantillas de Procedimientos le permiten crear **Procedimientos Operativos Estándar (POE)**, listas de verificación de seguridad o flujos de trabajo de mantenimiento que pueden adjuntarse a las Órdenes de Trabajo. Guían a los técnicos paso a paso y capturan datos estructurados.

## Guía de referencia: Tipos de entrada de tareas

| Tipo de entrada | Descripción | Mejor caso de uso |
|---|---|---|
| **Campo de texto** | Permite al técnico escribir una respuesta de texto breve. | Registrar números de serie, nombres de piezas o notas de observación. |
| **Casilla de verificación** | Casilla de alternancia simple para confirmación de un solo elemento. | Verificaciones visuales básicas (ej.: «Protecciones en su lugar», «Área limpiada»). |
| **Campo numérico** | Limita estrictamente la entrada a valores numéricos. | Registrar lecturas de medidores, temperaturas, PSI o niveles de voltaje. |
| **Elección única** | Presenta una lista personalizada de opciones, obligando al técnico a seleccionar exactamente una respuesta. | Condiciones de estado (ej.: «Aprobado / Necesita reparación / Fallido»). |
| **Lista de verificación** | Presenta una lista personalizada de opciones, permitiendo seleccionar múltiples respuestas. | Verificaciones de múltiples partes. |
| **Inspección** | Formato de entrada de condición estandarizado para inspecciones de activos. | Auditorías periódicas del estado físico de activos. |
| **Sí-No-N/A** | Proporciona tres botones de selección rápida preestablecidos. | Preguntas de cumplimiento o seguridad estándar. |
| **Fecha** | Abre un selector de calendario para registrar un día específico. | Registrar fechas de vencimiento o instalación de piezas. |
| **Fecha y hora** | Abre un selector de calendario y reloj. | Registrar el momento exacto en que ocurrió un incidente. |
